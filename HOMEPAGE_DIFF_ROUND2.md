# Homepage visual diff – round 2 (mobile-focused)

## 1. Hero section
Before:
- Hero copy sat in a relatively plain split layout.
- CTA buttons worked, but on mobile they visually behaved like standard buttons rather than a decisive first action.
- Supporting proof points were present, but their mobile rhythm was weaker.

After:
- Mobile padding tightened and headline scale tuned for smaller screens.
- Primary and secondary CTA now expand to full width on mobile.
- Proof cards are arranged as 2 + 1 on mobile, which reads faster and wastes less vertical space.
- Mascot detail cards switch to two columns earlier on larger phones.

## 2. “How it works” strip
Before:
- Mobile version collapsed into a vertical stack, which increased page length and reduced scan speed.

After:
- Mobile version now uses a compact 3-card grid immediately visible above the fold continuation.
- Each step is easier to compare at a glance.

## 3. Certificate preview section
Before:
- Good functional block, but mobile spacing was slightly heavy.
- Template selector was stacked and visually long on mobile.
- CTA buttons were not taking full mobile width.

After:
- Section padding tightened for mobile.
- Template selector now becomes a horizontal swipe row on mobile.
- Buy/gift CTA buttons are full width on mobile.
- Preview shell keeps a cleaner frame with less surrounding visual noise.

## 4. Membership / pricing section
Before:
- Cards stacked vertically on mobile, creating a long scroll and weaker comparison.
- Product comparison required too much reading distance.

After:
- Cards become a horizontal swipe comparison rail on mobile.
- Added a short mobile hint explaining the interaction.
- Card internals were tuned for mobile: less dead space, more stable title/price rhythm.

## 5. Real impact stats
Before:
- Already reasonably strong.

After:
- Minor spacing tightening only; section left largely intact because it already worked well on mobile.

## 6. Gifting use-cases
Before:
- Mobile version was longer and more list-like.

After:
- Converted to a 2-column mobile grid.
- Faster scan, better rhythm, shorter scroll depth.
- Gift and Wanted CTA buttons now expand full width on mobile.

## 7. Reviews
Before:
- Reviews stacked vertically on mobile.

After:
- Reviews now use a swipeable horizontal rail on mobile.
- Better perceived polish, less homepage fatigue, stronger social proof density.

## 8. FAQ teaser
Before:
- Standard 3-card layout that became a long vertical stack on mobile.

After:
- FAQ cards now also behave as a swipeable mobile rail.
- This preserves visibility without pushing the footer too far down.

## 9. Header + mobile CTA
Before:
- Header was functional but visually neutral.
- Sticky CTA worked, but felt more like a utility bar than a premium purchase prompt.
- Main content bottom padding was borderline for the sticky CTA height.

After:
- Header padding and controls were refined for smaller screens.
- Mobile menu buttons got clearer touch targets and more finished styling.
- Sticky CTA is now framed as a premium floating purchase module with a trust note.
- Main page bottom padding increased so the sticky CTA does not crowd the last content.

## 10. Template selector (global impact)
Before:
- Mobile selector was vertically long.

After:
- Mobile selector now scrolls horizontally with snap behavior.
- This improves both homepage preview and purchase flow.
